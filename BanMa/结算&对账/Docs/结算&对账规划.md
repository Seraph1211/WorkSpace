### 结算单规划
- 用途：
	- 「店铺」申请提现，审批通过后推送「结算单」到财务工作台处理
	- 处理「售后」退款，客服进行确认退款操作后，推送「结算单」到财务工作台处理
- 如何处理批量结算？

### 财务结算场景：
- 给「店铺」结算 -> 按「结算单」批量结算
- 处理「售后」退款 -> 按「结算单」批量结算
- 内部结算（分给平台和包赔）-> 根据「财务月报」按月结算
	- 给平台 -> 扣除支付服务费后，按月结算
	- 给包赔 -> 按月结算



### 财务对账要求
- 按月生成月度财务报表，该报表以平台实际销售流水为数据源，与「结算单」无关
- 月报中的数据以订单为最细颗粒度，每条记录除了订单的基本信息之外，至少需要再包含：店铺名、用户实付、店铺应结算、平台服务费、支付服务费、履约包赔
- 财务按照月报完成报税、平台&包赔结算等动作



### 店铺账户资金构成
- 正向资金
	- 待入账
	- 可提现
	- 提现中
	- 已提现
- 负向资金
	- 待冲抵

- 正向资金用于记录店铺下商品销售后的收入
- 负向资金用于记录店铺商品产生售后，下次提现需要冲抵的金额
	- 这部分资金，用于记录



### 售后资金的处理
- 售后评估，如果需要向客户赔付，平台直接垫付，创建「结算单」，财务根据结算单打款
- 财务打款后，该部分结算单金额直接计入店铺的「待冲抵」
- 当店铺下一次提款时，自动抵扣待冲抵部分金额
