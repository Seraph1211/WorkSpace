# 斑马优号数据看板 PRD

---

## 一、我们要解决什么问题

### 1.1 现状

斑马优号是游戏账号交易平台，覆盖 Web、H5、APP 三个端（当前重点为 H5），业务分为**买号**和**卖号**两条线。流量主要来自自媒体短视频切片，绝大多数导入 H5。

**6 月数据（6.1 - 6.10，共 10 天）：**

| 指标 | 累计 | 日均 |
|------|------|------|
| 活跃用户数 | 19,332 | ~1,933 |
| 咨询数 | 15,389 | ~1,539 |
| 销售订单数 | 371 | ~37 |

> **当前数据口径说明：**
> - 活跃用户数：每日首页 UV 相加（同一用户不同天来访计入多次）
> - 咨询数：每日咨询次数相加（同一用户同一天既发起买号咨询又发起卖号咨询，计为 2）
> - 销售订单数：每日交易成功订单数相加
>
> 上述口径可能存在不合理之处，将在第三章「指标计算口径」中统一规范。

- 首页→咨询转化率：**79.6%**（15,389 / 19,332）
- 咨询→交易转化率：**2.4%**（371 / 15,389）
- 首页→交易转化率：**1.9%**（371 / 19,332）

**核心事实：用户咨询意愿极强（~80%），但极少走到交易（~2.4%）。流失的主战场是咨询→交易这一段。**

**业务特点：**

- 用户在查看账号详情后发起咨询时才需注册（手机号），下单不强制咨询
- 但游戏账号换绑必须由客服协助完成，用户最终一定会和客服产生接触
- 少数换绑跨天完成（行业常态）
- 存在账号被他人抢先买走的情况（目前不频繁）

### 1.2 核心矛盾

**矛盾不在"用户来不来"，而在"用户来了为什么不买"。**

平台有着极高的咨询意愿（~80% 的活跃用户会发起咨询），说明两件事：

1. 流量质量不差——用户是真的有账号交易需求才来的
2. 首页和列表页的吸引力足够——用户愿意迈出咨询这一步

但从咨询到交易，转化率断崖式下跌到 2.4%。这意味着在「用户表达购买意向」到「完成交易」之间，存在系统性的阻塞。

**可能的原因方向（待数据验证）：**

- 客服接了咨询但没能促成下单（匹配不准、响应慢、话术问题）
- 用户想买的账号实际不可得（已被买走、价格谈不拢）
- 下单到支付之间有摩擦（支付方式、流程体验）
- 支付后换绑等待过长导致用户放弃（尤其是跨天场景）

**当前最大的信息盲区：** 咨询用户中，到底有多少人走到了下单？多少人支付了？多少人卡在换绑？这些中间环节的数据没有拉通，导致无法定位具体是哪个环节出了问题。

### 1.3 本期目标与边界

**核心目标：** 建立从「首页进入 → 交易完成」的全流程转化漏斗，精准定位用户流失最大的环节。其中，咨询→交易这一段转化率极低（2.4%），是本期重点关注对象。

**具体目标：**

1. 展示首页到交易完成的完整漏斗，看清每一步的转化和流失
2. 拉通咨询后的隐形链路（咨询→下单→支付→换绑→完成），定位断崖点
3. 买号、卖号两条业务线独立追踪
4. 分平台（Web / H5 / APP）对比，重点关注 H5
5. 支持按新老用户、游戏品类交叉分析

**使用角色：** 产品经理、运营人员
**数据更新：** T+1

## 二、用户转化路径（统一抽象）

### 2.1 买号路径

用户到达买号咨询有三条路径，所有路径在「买号咨询」处汇合，之后的链路完全一致。

**路径一（标准路径——浏览详情后咨询）：**

```
首页 → 账号列表 → 账号详情 → 买号咨询 → 下单 → 支付成功 → 交易完成
```

用户在平台内自主浏览，看完账号信息后联系客服。

**路径二（首页直接咨询）：**

```
首页 → 买号咨询 → 下单 → 支付成功 → 交易完成
```

用户不浏览列表，进入首页后直接找客服要号。

**路径三（列表直接咨询）：**

```
首页 → 账号列表 → 买号咨询 → 下单 → 支付成功 → 交易完成
```

用户在列表页通过悬浮按钮直接联系客服，不进入详情页。

**漏斗设计思路：**

三条路径在「买号咨询」处汇合，所以漏斗分两层处理：

- **核心漏斗（看板展示主体）：** 以「买号咨询」为起点，展示 咨询 → 下单 → 支付成功 → 交易完成。这是所有用户必经的统一链路，也是 2.4% 转化率倒下的地方。
- **汇合前分流（在页面分析中体现）：** 首页 → 账号列表 → 账号详情这一段放在看板 3（页面分析），通过页面 PV/UV 和咨询入口分布来说明用户从哪发起咨询，不硬塞进统一漏斗。

### 2.2 卖号路径

卖号业务本期不设漏斗，仅关注两个指标：卖号咨询用户数、各游戏的卖号咨询分布、每个商家的咨询分布。

### 2.3 各环节流失原因（穷举）

**咨询 → 下单（核心流失段）**

- 用户想要的账号已被别人买走
- 客服推荐了不合适/不满意的账号
- 价格谈不拢（用户预期价格和实际售价差距大）
- 客服响应太慢，用户等待中流失
- 用户只是随便问问，本身没有明确的购买意图
- 客服未回复/漏回复

**下单 → 支付成功**

- 支付方式不支持或不够便捷
- 支付页面跳转异常或加载慢
- 用户下单后反悔
- 订单金额异常

**支付成功 → 交易完成**

- 换绑等待时间过长（尤其是跨天），用户放弃
- 换绑过程中出现问题导致退款
- 用户支付后反悔发起退款
- 账号实际状态与描述不符，导致换绑失败

## 三、我们要看哪些指标

### 3.1 一级核心指标

一级指标是每日必看、反映平台健康度的核心数据。

**流量指标：**

| 指标 | 说明 |
|------|------|
| 首页 UV | 当日首页独立访客数 |
| 首页 PV | 当日首页浏览总次数 |

**转化指标（买号）：**

| 指标 | 说明 |
|------|------|
| 买号咨询 UV | 当日发起买号咨询的独立用户数 |
| 下单数 | 当日提交的买号订单数 |
| 支付成功数 | 当日支付成功的订单数 |
| 交易成功数 | 当日交易完成的订单数 |
| 整体转化率 | 交易成功数 / 首页 UV |
| 咨询→交易转化率 | 交易成功数 / 买号咨询 UV |

**收入指标：**

| 指标 | 说明 |
|------|------|
| GMV | 当日买号交易总额 |
| 客单价 | GMV / 交易成功数 |

> GMV 和客单价互为补充：GMV 跌了，拆到「交易成功数」和「客单价」两个因子，判断是订单量问题还是单价问题。

**卖号指标：**

| 指标 | 说明 |
|------|------|
| 卖号咨询 UV | 当日发起卖号咨询的独立用户数 |

### 3.2 二级诊断指标

当一级指标异常时，用二级指标下钻定位原因。

**环节转化率（拆解整体转化率）：**

| 指标 | 作用 |
|------|------|
| 咨询→下单率 | 定位客服促成效率 |
| 下单→支付率 | 定位支付流程问题 |
| 支付→交易率 | 定位换绑/售后问题 |

**页面分析指标：**

| 指标 | 作用 |
|------|------|
| 各页面 PV / UV | 了解流量分布 |
| 页面退出率 | 哪个页面在赶客 |
| 人均停留时长 | 判断用户是在看还是在纠结 |
| 咨询入口来源分布 | 用户从哪发起的咨询 |

**客服指标：**

| 指标 | 作用 |
|------|------|
| 客服首次响应时长（中位数） | 判断响应速度是否是转化低的原因 |
| 未回复咨询数 | 判断是否有漏单 |

**多维下钻：**

| 维度 | 作用 |
|------|------|
| 分游戏品类 | 哪个游戏的转化拖了后腿 |
| 分用户类型（新 / 老） | 不同人群的流失节点差异 |
| 分价格区间 | 低价号好卖还是高价号好卖 |
| 分平台（Web / H5 / APP） | 不同端的转化差异 |

### 3.3 指标计算口径

**用户类：**

| 指标 | 计算口径 |
|------|---------|
| 未注册用户 | 历史上从未发起过咨询（即未注册）的设备 ID |
| 已注册用户 | 历史上发起过至少一次咨询（即已注册手机号）的设备 ID |

> 注册发生在首次发起咨询时，因此「未注册 / 已注册」等价于「从未咨询 / 已咨询」。

**PV/UV 类：**

| 指标 | 计算口径 |
|------|---------|
| 页面 PV | 页面被打开/刷新的次数 |
| 页面 UV | 访问页面的独立设备 ID 数，当日去重 |
| 首页 UV | 访问首页的独立设备 ID 数，当日去重 |

**咨询类：**

| 指标 | 计算口径 |
|------|---------|
| 总咨询 UV | 当日发起过咨询（买号或卖号）的独立用户数，去重 |
| 买号咨询 UV | 当日发起过买号咨询的独立用户数，去重 |
| 卖号咨询 UV | 当日发起过卖号咨询的独立用户数，去重 |

> 买号咨询 UV + 卖号咨询 UV ≥ 总咨询 UV。同一用户同一天既发起买号咨询又发起卖号咨询时，两边各计 1，总咨询计 1。

**交易类：**

| 指标 | 计算口径 |
|------|---------|
| 下单数 | 提交订单的次数（含未支付），同日同用户多次下单计多次 |
| 支付成功数 | 支付回调成功的订单数 |
| 交易成功数 | 换绑完成且未退款的订单数 |
| GMV | 交易成功订单的金额总和 |
| 客单价 | GMV / 交易成功数 |

**转化率类：**

| 指标 | 公式 |
|------|------|
| 咨询→下单率 | 下单数 / 买号咨询 UV |
| 下单→支付率 | 支付成功数 / 下单数 |
| 支付→交易率 | 交易成功数 / 支付成功数 |
| 整体转化率 | 交易成功数 / 首页 UV |
| 咨询→交易转化率 | 交易成功数 / 买号咨询 UV |

**时间归属：** 浏览、下单、支付、交易均按事件发生日归属。

### 3.4 贯穿始终的三个分析维度

所有看板支持以下三个维度交叉分析：

| 维度 | 选项 | 说明 |
|------|------|------|
| 平台 | Web / H5 / APP | 默认 H5 |
| 用户类型 | 未注册用户 / 已注册用户 | 等价于"从未咨询 / 已咨询" |
| 游戏品类 | 全部 / 单游戏 | 按具体游戏拆分 |
