# 斑马优号数据看板产品需求文档（PRD）


## 一、项目背景与目标

### 1.1 背景
斑马优号作为游戏账号交易平台，目前H5端日浏览UV约1-2万，但整体转化率仅0.15%-0.23%，远低于行业平均水平。

当前核心痛点：
- 不知道用户在哪个环节流失、为什么流失
- 当前数据统计口径不清晰，缺乏完善的用户行为数据采集体系
- 产品优化和运营策略缺乏数据支撑

### 1.2 核心目标
**最核心目标：精准定位用户流失点**，通过数据驱动产品优化，提升整体转化率

**具体目标：**
1. 分端对数据进行统计（Web、H5、APP）
2. 清晰展示用户从首页浏览到交易完成的全链路转化漏斗
3. 区分新老用户的转化路径差异，找到不同人群的流失节点
4. 掌握每个页面的PV/UV表现，定位页面级问题
5. 量化客服转化效率，支撑运营优化
6. 了解咨询入口分布，优化用户决策路径

### 1.3 范围说明
- **本期范围**：H5端数据看板 V2.0
- **数据更新**：T+1（每日凌晨更新前一天数据）
- **使用角色**：产品经理、运营人员

## 二、核心概念与统计口径

### 2.1 新老用户定义

| 用户类型 | 统计口径             | 备注                         |
| ---- | ---------------- | -------------------------- |
| 新用户  | 首次访问平台的用户        | 未注册用户，以设备ID为主要标识，注册后关联用户ID |
| 老用户  | 之前已访问过，本次再次来访的用户 | 已注册用户，设备ID或用户ID在历史数据中已有记录  |

### 2.2 PV与UV定义

| 指标        | 统计口径       | 备注                     |
| --------- | ---------- | ---------------------- |
| PV（页面浏览量） | 页面被打开的次数   | 用户每打开/刷新一次页面，计为1次PV    |
| UV（独立访客数） | 访问页面的独立用户数 | 按设备ID去重，同一用户多次访问只算1次UV |

### 2.3 交易/接待相关定义

| 指标      | 统计口径         | 备注            |
| ------- | ------------ | ------------- |
| 支付成功订单数 | 完成支付的订单数量    | 以后端支付回调为准     |
| 交易成功订单数 | 换绑且交易完成的订单数  | 退款/取消订单不计入    |
| 客服接待人数  | 发起客服咨询的独立用户数 | 同一用户多次咨询只算1人  |
| 客服转化率   | 咨询后成功下单的用户比例 | 下单用户数 / 咨询用户数 |

### 2.4 用户转化路径（全量）

用户从进入到最终交易完成，存在多条路径：

**主路径（标准路径）：**
```
首页 → 选游戏 → 账号列表 → 账号详情 → 咨询客服 → 下单 → 支付 → 交易完成
```

**旁路路径（常见）：**
```
首页 → 直接咨询客服 → 下单 → ...
首页 → 搜索 → 账号详情 → 咨询 → ...
首页 → 个人中心 → 我的订单 → ...
```

## 四、看板模块详细设计

### 看板1：总览看板

**定位：** 一眼看清整体转化情况和各环节流失概貌

#### 模块1：核心指标卡片（顶部）

一行8个核心数据卡片，每个卡片展示：当前值 + 环比昨日变化%

| 序号  | 指标名称    | 说明           |
| --- | ------- | ------------ |
| 1   | 浏览UV    | 当日独立访客数      |
| 2   | 浏览PV    | 当日页面浏览总次数    |
| 3   | 咨询用户数   | 发起客服咨询的用户数   |
| 4   | 注册用户数   | 新注册用户数       |
| 5   | 下单订单数   | 提交的订单数       |
| 6   | 支付成功订单数 | 完成支付的订单数     |
| 7   | 交易成功订单数 | 确认收货的订单数     |
| 8   | 整体转化率   | 交易成功数 / 浏览UV |

---


### 看板2：全流程转化与流失
#### 模块1：全链路转化漏斗（核心）

展示从浏览到交易完成的关键节点漏斗，支持新老用户筛选、游戏筛选，支持节点展示/隐藏、支持按平台筛选（Web/H5/APP）

主页UV -> 游戏列表UV -> 商品列表UV -> 发起咨询UV -> 提交订单数 -> 支付成功数 -> 交易成功数


**交互说明：**
- 鼠标悬停显示：环节人数、上一环节转化率、累计转化率、流失人数
- 支持切换"全部/新用户/老用户"查看不同人群的漏斗；支持节点展示/隐藏的设置、按
- 点击环节可下钻到漏斗分析看板的对应环节

#### 模块2：流失严重程度排行

按流失人数从多到少排序，快速定位最大流失点（根据配置的展示节点自动更新流失节点）

| 排名  | 流失节点    | 流失人数   | 流失率 | 影响程度  |
| --- | ------- | ------ | --- | ----- |
| 1   | 浏览→查看详情 | 11,250 | 75% | 🔴 严重 |
| 2   | 详情→咨询   | 2,550  | 68% | 🔴 严重 |
| 3   | 咨询→下单   | 900    | 75% | 🔴 严重 |
| 4   | 下单→支付   | 60     | 20% | 🟡 一般 |
| 5   | 支付→换绑   | 24     | 10% | 🟢 良好 |
| 6   | 换绑→确认   | 11     | 5%  | 🟢 良好 |

流失率 = （上一环节人数 - 本环节人数）/ 上一环节人数

#### 模块3：转化趋势图

- 近30天整体转化率趋势（折线图）
- 可切换查看各关键环节转化率趋势（多折线对比）

用户从哪些页面发起咨询，帮助理解用户决策路径
买号咨询：

| 咨询入口页面 | 咨询人数 | 占总咨询比例 | 该页面UV | 页面→咨询转化率 |

| 咨询入口页面  | 咨询人数 | 占总咨询比例 | 该页面UV | 页面→咨询转化率 |

**业务价值：**
- 详情页咨询占比高 → 用户是看完账号后有兴趣才咨询（精准需求）

- 不同入口的后续转化率差异，可反推用户质量和需求强度
