# 斑马优号数据看板 PRD V3

> 本版采用「看板驱动」组织：先列清楚要做哪几张看板，再逐一定义每张看板看什么数据、怎么展示、怎么交互。

---

## 一、看板清单总览


| 编号  | 看板名称    | 定位                 | 受众    | 核心问题                          |
| --- | ------- | ------------------ | ----- | ----------------------------- |
| 1   | 核心指标总览  | 登录后默认首页，一眼看平台整体健康度 | 全员    | 今天平台怎么样？有没有需要关注的异常？           |
| 2   | 买号转化看板  | 全链路漏斗诊断，定位流失环节及原因  | 产品、运营 | 用户从进来到成交，哪一步漏得最多？不同品类/平台差异在哪？ |
| 3   | 页面与流量分析 | 各页面流量表现 + 首页流量流向分析 | 产品、运营 | 每个页面承接了多少流量？用户从首页去了哪里？        |
| 4   | 销售客服看板  | 客服维度拆解交易转化，评估人效    | 运营、管理 | 每个客服转化效率如何？哪个环节耗时最长？          |


### 看板间关系

```
看板 1（总览）
  ├─ 转化率/漏斗 → 看板 2（转化看板）
  │     ├─ 售前环节（1~4层）→ 看板 3（页面与流量分析）
  │     └─ 交易环节（5~8层）→ 看板 4（销售客服看板）
  ├─ 活跃用户数 → 看板 3（页面与流量分析）
  └─ GMV/订单数 → 看板 4（销售客服看板）
```

### 用户分类

> 平台用户按注册状态分为三类，各看板中涉及「用户」维度的分析均基于此分类。


| 分类    | 定义                          | 典型行为                     |
| ----- | --------------------------- | ------------------------ |
| 未注册用户 | 当日产生浏览行为但未注册账号的用户           | 浏览首页、列表、详情，未发起注册即离开      |
| 新注册用户 | 当日产生浏览行为并完成注册的用户            | 浏览 → 注册 → 可能进一步咨询或交易     |
| 老用户   | 注册时间早于当日的用户（无论当日是否有浏览/咨询行为） | 已注册身份访问，可直接咨询，无需再次经过注册环节 |


> **判定方式（推荐）：** 数仓 ETL 时用 user_id 关联用户注册表，按「注册日期 = 事件日期」判定新注册用户，「注册日期 < 事件日期」判定老用户，「无 user_id」判定未注册用户。埋点侧无需额外改造。

## 二、看板 1：核心指标总览

> **定位：** 登录看板系统后默认展示的第一屏，一眼看今日平台整体健康度，所有人可读。核心问题：今天平台怎么样？有没有需要立刻关注的异常？

### 2.1 核心指标及口径

#### 2.1.1 流量指标


| 指标名          | 计算口径                |
| ------------ | ------------------- |
| 活跃用户数（首页 UV） | 当日访问首页的独立设备 ID 数，去重 |
| 首页 PV        | 当日首页被打开/刷新的总次数      |
| 新注册用户数       | 当日完成注册的独立用户数，去重     |
| 买号咨询 UV      | 当日发起过买号咨询的独立用户数，去重  |
| 卖号咨询 UV      | 当日发起过卖号咨询的独立用户数，去重  |


#### 2.1.2 交易与收入指标


| 指标名      | 计算口径                          |
| -------- | ----------------------------- |
| 订单数（下单数） | 当日提交买号订单的次数，含未支付，同日同用户多次下单计多次 |
| 下单 UV    | 当日提交过买号订单的独立用户数，去重            |
| 支付成功数    | 当日支付回调成功的订单数                  |
| 支付成功 UV  | 当日支付回调成功的独立用户数，去重             |
| 交易成功数    | 当日换绑完成且未退款的订单数                |
| 交易成功 UV  | 当日换绑完成且未退款的独立用户数，去重           |
| GMV      | 当日交易成功订单的金额总和                 |
| 客单价      | GMV / 交易成功数（交易成功数为 0 时显示"—"）  |


#### 2.1.3 买号咨询转化漏斗

> 漏斗展示 4 层 UV 数量，层与层之间标注转化率。


| 漏斗层级    | 指标      | 说明                |
| ------- | ------- | ----------------- |
| 第 1 层   | 买号咨询 UV | 当日发起过买号咨询的独立用户数   |
| 1→2 转化率 | 下单转化率   | 下单 UV / 买号咨询 UV   |
| 第 2 层   | 下单 UV   | 当日提交过买号订单的独立用户数   |
| 2→3 转化率 | 支付转化率   | 支付成功 UV / 下单 UV   |
| 第 3 层   | 支付成功 UV | 当日支付回调成功的独立用户数    |
| 3→4 转化率 | 交易成功转化率 | 交易成功 UV / 支付成功 UV |
| 第 4 层   | 交易成功 UV | 当日换绑完成且未退款的独立用户数  |
| 底部汇总    | 买号整体转化率 | 交易成功 UV / 买号咨询 UV |


#### 2.1.4 趋势折线图

用户可自行配置要展示的指标和时间维度：

- **可选指标池**：2.1.1~2.1.3 中全部 17 个指标均可勾选
- **默认展示**：GMV、活跃用户数、买号咨询 UV、订单数、买号整体转化率（5 条线，首次加载的出厂设置）
- **指标配置入口**：图表右上角「⚙ 配置」按钮，弹出多选下拉，已选指标显示在图中
- **时间维度**：近 7 天 / 近 30 天 / 自定义范围（日期选择器），切换后保留当前指标选择
- **Y 轴策略**：系统自动检测已选指标的量纲——若同时包含数值类（金额/人数）和百分比类，自动启用双 Y 轴；纯数值类则单 Y 轴

## 三、看板 2：买号转化看板

### 3.1 展示的数据指标及口径

#### 3.1.1 交易全流程

> 看板 2 覆盖买号交易完整链路，共 8 个环节：

```
访问首页 → 选择游戏 → 浏览列表 → 查看详情 → 发起咨询 → 提交下单 → 完成支付 → 交易成功
```

> 其中前 4 个环节（访问首页 ~ 查看详情）为售前浏览阶段，后 4 个环节（发起咨询 ~ 交易成功）为交易阶段。看板 1 已覆盖交易阶段的漏斗快照，看板 2 补充售前浏览阶段，并提供分维度拆解分析能力。

#### 3.1.2 各环节 UV 指标


| 序号  | 环节   | 指标名     | 计算口径                     | 数据状态   |
| --- | ---- | ------- | ------------------------ | ------ |
| 1   | 访问首页 | 首页 UV   | 当日访问首页的独立设备 ID 数，去重      | ✅ 已有   |
| 2   | 选择游戏 | 选游戏 UV  | 当日在首页点击/选择了具体游戏的独立用户数，去重 | ⚠️ 待确认 |
| 3   | 浏览列表 | 列表页 UV  | 当日进入账号列表页的独立用户数，去重       | ⚠️ 待确认 |
| 4   | 查看详情 | 详情页 UV  | 当日进入账号详情页的独立用户数，去重       | ⚠️ 待确认 |
| 5   | 发起咨询 | 买号咨询 UV | 当日发起过买号咨询的独立用户数，去重       | ✅ 已有   |
| 6   | 提交下单 | 下单 UV   | 当日提交过买号订单的独立用户数，去重       | ✅ 已有   |
| 7   | 完成支付 | 支付成功 UV | 当日支付回调成功的独立用户数，去重        | ✅ 已有   |
| 8   | 交易成功 | 交易成功 UV | 当日换绑完成且未退款的独立用户数，去重      | ✅ 已有   |


#### 3.1.3 环节间转化率


| 环节间         | 指标名        | 计算口径              |
| ----------- | ---------- | ----------------- |
| 首页 → 选择游戏   | 首页→选游戏转化率  | 选游戏 UV / 首页 UV    |
| 选择游戏 → 浏览列表 | 选游戏→列表转化率  | 列表页 UV / 选游戏 UV   |
| 浏览列表 → 查看详情 | 列表→详情转化率   | 详情页 UV / 列表页 UV   |
| 查看详情 → 发起咨询 | 详情→咨询转化率   | 买号咨询 UV / 详情页 UV  |
| 发起咨询 → 提交下单 | 咨询→下单转化率   | 下单 UV / 买号咨询 UV   |
| 提交下单 → 完成支付 | 下单→支付转化率   | 支付成功 UV / 下单 UV   |
| 完成支付 → 交易成功 | 支付→交易成功转化率 | 交易成功 UV / 支付成功 UV |


#### 3.1.4 环节间流失


| 环节间         | 指标名        | 计算口径              |
| ----------- | ---------- | ----------------- |
| 首页 → 选择游戏   | 首页→选游戏流失数  | 首页 UV - 选游戏 UV    |
| 选择游戏 → 浏览列表 | 选游戏→列表流失数  | 选游戏 UV - 列表页 UV   |
| 浏览列表 → 查看详情 | 列表→详情流失数   | 列表页 UV - 详情页 UV   |
| 查看详情 → 发起咨询 | 详情→咨询流失数   | 详情页 UV - 买号咨询 UV  |
| 发起咨询 → 提交下单 | 咨询→下单流失数   | 买号咨询 UV - 下单 UV   |
| 提交下单 → 完成支付 | 下单→支付流失数   | 下单 UV - 支付成功 UV   |
| 完成支付 → 交易成功 | 支付→交易成功流失数 | 支付成功 UV - 交易成功 UV |


#### 3.1.5 分维度拆解

> 上述全部指标（UV、转化率、流失数）均支持按以下维度拆解分析：


| 维度   | 说明                             | 数据状态   |
| ---- | ------------------------------ | ------ |
| 游戏品类 | 按游戏拆开漏斗（王者荣耀、和平精英等），对比各品类转化差异  | ⚠️ 待确认 |
| 平台   | 按 Web / H5 / APP 拆开漏斗，对比各端转化差异 | ⚠️ 待确认 |


#### 3.1.6 咨询相关指标

> 用于分析注册门槛对咨询环节的影响，区分新老用户的咨询行为。


| 指标名        | 计算口径                             |
| ---------- | -------------------------------- |

| 指标名   | 计算口径                         |
| ----- | ---------------------------- |
| 下单转化率 | 下单数 / 咨询人数（咨询人数为 0 时显示"—"）   |
| 支付转化率 | 支付成功数 / 下单数（下单数为 0 时显示"—"）   |
| 成交转化率 | 成交数 / 支付成功数（支付成功数为 0 时显示"—"） |


#### 5.1.4 时效类指标


| 指标名       | 计算口径                                             |
| --------- | ------------------------------------------------ |
| 咨询→下单平均耗时 | 从发起咨询到提交下单的平均时间间隔（仅计算当天咨询且当天成功下单的用户，取该客服经手订单的均值） |
| 下单→支付平均耗时 | 从提交下单到支付回调成功的平均时间间隔（取该客服经手且当天支付的订单均值）            |
| 支付→成交平均耗时 | 从支付成功到换绑完成的平均时间间隔（取该客服经手且当天成交的订单均值）              |


#### 5.1.5 客服维度

> 上述全部指标均支持按单个客服查看或所有客服汇总查看。客服明细表每个客服一行，包含全部 11 个指标。
